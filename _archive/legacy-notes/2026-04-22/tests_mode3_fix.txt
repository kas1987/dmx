Signs are now bit-packed (8 per byte) in a separate array.
This fixes uint8 overflow at M=8 and zero-value sign loss at M=6/7.

Key improvements from the fix:
- M=8 now works correctly (was completely broken — sign<<8 overflowed uint8)
- Sign bits are preserved independently of mantissa truncation
- Negligible overhead: 1 bit per value = N/8 bytes for sign stream
- compressed_bytes property includes sign stream size
- dmx_cli.py untouched — no API changes to public functions
